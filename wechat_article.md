# 我用 AI 做了一个"自动化股票研究员"——从数据搜集到投资报告，全程无需人工干预

> 一个开源工具，10个研究维度，自动生成一份媲美买方研报的深度分析报告。

---

## 先说一个真实场景

你想研究一只股票，比如某家港股的消费品公司。

你会怎么做？

打开Bloomberg，查财务数据。再去SEC看年报。搜英文新闻，再搜中文新闻。找可比公司，手动对比估值。整理行业报告，归纳竞争格局……

如果认真做，这套流程下来，**快则两天，慢则一周**。

我最近做了一个工具，把上面这套流程完整地自动化了。

输入一个股票代码，它自己去搜资料、读财报、整理数据、写报告，最后给出一个 BUY / HOLD / SELL 的投资判断。

整个过程不需要你做任何事情。

---

## 它长什么样

给你看一下最终输出的报告结构：

**Part 1：核心主线**
行业在这段时间内发生了什么变化？公司是怎么应对的？市场当时的预期是什么，后来验证了吗？

**Part 2：行业全景**
行业市场规模、增速、竞争格局、进入壁垒——这是做任何行业研究绕不开的底层框架。

**Part 3：可比公司研究**
选取2-4家历史更悠久的可比公司，复盘他们怎么走过来的，踩过哪些坑，对我们研究的公司有什么启示。

**Part 4：公司基本面**
历史沿革、核心产品、竞争优势（有数据论证）、市场份额变化。

**Part 5：财务全景**
收入、利润、毛利率、ROE、ROIC、杜邦拆解、现金流质量——把财务数据真正讲清楚。

**Part 6：产能与资本开支**
CAPEX怎么花的，产能利用率如何，在建项目情况。

**Part 7：治理与股东回报**
管理层历史承诺有没有兑现，分红回购情况，激励机制是否与股东对齐。

**Part 8：增长逻辑**
每个阶段的增长drivers是什么，有哪些已经消退，未来的天花板在哪里。

**Part 9：股价复盘**
历史上每一轮大涨大跌的驱动因素，是基本面驱动、情绪驱动还是资金驱动。

**Part 10：估值方法论与买卖点**
这个行业该怎么估值，历史估值区间是什么，什么时候是好的买点和卖点。

**Part 11：综合判断**
通读整份报告后，给出 BUY / HOLD / SELL 信号，附上置信度和核心假设。

---

一份报告覆盖 **11 个维度**，**平均超过 3 万字**，图文并茂，全程用数据说话。

这不是 ChatGPT 随便总结的那种泛泛而谈——每一条结论背后都有明确的数据来源，搜索过程全部可溯源。

## 它是怎么工作的

这个工具的核心是一个**4阶段的 Agentic 流程**：

```
阶段1：Research Loop（研究循环）
    ↓
AI 自主调用工具搜集数据，迭代直到信息充分
    ↓
阶段2：Extraction（数据提取）
    ↓
轻量模型从原始数据中提炼结构化摘要
    ↓
阶段3：Writing（报告撰写）
    ↓
高能力模型按照研报格式写出完整章节
    ↓
阶段4：Judgment（综合判断）
    ↓
最终给出 BUY / HOLD / SELL + 置信度
```

### 它能用哪些工具搜数据？

- **`web_search`** — DuckDuckGo 全网搜索，中英文都支持
- **`get_stock_info`** — 获取股票基本面数据（市值、PE、PB、毛利率等）
- **`get_financials`** — 拉取年度/季度财务报表（利润表、资产负债表、现金流）
- **`get_sec_filings`** — 读取 SEC EDGAR 原始年报（适用于美股）
- **`read_url`** — 抓取任意网页内容（新闻、研报、公告等）
- **`read_local_file`** — 读取本地 PDF 或 Markdown 文件

它每次研究一个 Part，会反复调用这些工具，**直到认为数据足够为止**。平均每个 Part 调用 4-8 次工具，整个报告下来大概 60-100 次 API 调用。

## 两个我最在意的设计

### 1. Preview 模式——先确认对象，再深度研究

做研究最怕的是什么？

跑了两个小时，发现研究的是同名的另一家公司。

我加了一个设计：**程序启动后，先只研究"公司基本介绍"这一节（Part 4）**，把结果打印出来给你看，让你确认：

```
--- PREVIEW: 公司基本面 ---
## Part 4: 公司基本面

### 1. 公司基本介绍
乐舒适（2698.HK）成立于...

Preview completed. Does this look OK? Continue with full research? [y/N]:
```

你确认没问题，输入 `y`，才开始跑完整的研究。

这一步虽然简单，但能节省很多冤枉时间。

---

### 2. Checkpoint 断点续传——跑到一半出问题，直接接着来

深度研究一家公司，完整跑完可能需要**40-90分钟**，中途可能遇到网络超时、API限流、或者你突然要关电脑。

我设计了自动存档机制：**每完成一个 Part，立即保存进度**。

如果中断了，下次重新运行同一个命令，程序会自动识别进度，从断点继续，**已完成的部分不会重跑**。

```
[INFO] Checkpoint found: checkpoint_2698.HK_2026-03-17.json (Parts [1, 2, 3, 4])
[INFO] Part 1 already completed, skipping.
[INFO] Part 2 already completed, skipping.
...
[INFO] Researching Part 5: 财务全景
```

## 防幻觉机制

AI 做研究，最大的风险是**一本正经地编数据**。

我在 prompt 里加了专门的反幻觉规则：

> 「如果工具返回了"No results found"，你必须在报告里明确写出数据不可用，而不是自己推断或编造。所有分析必须严格基于工具成功返回的数据。」

同时加了**Target Lock（目标锁定）**：

> 「你正在研究的标的是 XXX。所有工具调用必须围绕这个标的。如果工具返回了其他公司的数据，你必须识别并忽略。在 web_search 时，搜索词中必须包含标的公司名称。」

这两条规则大幅减少了 AI "漂移"到其他公司或编造数据的情况。

## 模型配置

工具内置了一个**灵活的模型路由层**，你可以根据自己的 API 资源配置不同任务用不同模型：

| 任务 | 推荐级别 | 原因 |
|---|---|---|
| 研究循环（工具调用） | 中等模型 | 需要 function calling，但不需要最强推理 |
| 数据提取 | 轻量模型 | 结构化提取，速度优先 |
| 报告撰写 | 高能力模型 | 需要文笔和逻辑 |
| 最终判断 | 高能力模型 | 需要综合推理 |

目前默认配置是 **Gemini 3.1 Pro**（研究+写作）和 **Gemini 3 Flash**（提取）。也可以切换成 Claude（注释里有备用配置）。

- **输入 tokens**：约 50-200 万

## 如何使用

项目已开源：

**👉 https://github.com/Akkao114/research-agent**

**安装步骤：**

```bash
# 1. 克隆项目
git clone https://github.com/Akkao114/research-agent.git
cd research-agent

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置 API key
cp .env.example .env
# 编辑 .env，填入你的 API key

# 4. 运行
python main.py --mode deep 2698.HK
```

支持港股（`.HK`）、A股（`.SS`/.`SZ`）、美股（直接 ticker），以及**非上市公司关键词搜索**。

## 它能完全替代人吗？

不能。

它能帮你在一两个小时内完成**一个初级分析师需要几天才能做完的数据搜集和整理工作**。

但最终的判断，还是需要你自己思考。

AI 的综合判断（BUY/HOLD/SELL）是基于它搜集到的信息给出的，而它搜集到的信息可能是不完整的。网上没有的数据，它不知道。没有公开披露的内幕信息，它当然也不知道。

所以这个工具的正确用法是：**把它当作一个不知疲倦的初级研究员**，帮你做密集的数据搜集和初步框架整理。你负责拍板。
